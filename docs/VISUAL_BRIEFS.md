# Visual Briefs

These briefs translate the mood-board sprite sheet into production directions for launch visuals.

## Hero Visual Brief

Purpose: establish trust, intelligence, and creativity in the first viewport.

Composition:

- Gibson portrait or portrait-adjacent founder image.
- Systems collage: notes, wireframes, roadmap fragments, UI pieces, and project covers.
- Chrome/cosmic object as the editorial hero object.
- Black/white/paper foundation with restrained `signal` and `cosmic` accents.

Mood-board sources:

- Row 1: Gibson portrait variations.
- Row 2: systems artifact variations.
- Row 3: chrome/cosmic object variations.
- Row 5: section header composition.

Rules:

- Do not make the hero feel like a generic agency landing page.
- The portrait should feel intelligent and approachable, not overproduced.
- The collage should suggest system thinking, not clutter.
- If a realistic Gibson portrait is not ready, use an abstract systems collage until reviewed.

## Case Cover Brief

Purpose: make each flagship project feel like a premium product story.

Shared rules:

- Use black/white foundation.
- Let client-specific accent colors appear sparingly.
- Use large type and one strong visual object.
- Avoid unapproved private screenshots.
- Use public screenshots or abstracted mockups where needed.

### The Home Depot

- Theme: project calculators, material lists, input systems, omnichannel retail.
- Accent: restrained `solar` orange.
- Avoid: unapproved internal screens or performance claims.

### CarMax

- Theme: sell-your-car funnel, offer comprehension, testing systems.
- Accent: restrained yellow/blue nod without turning the page into CarMax branding.
- Public screenshots are acceptable.

### Mini Murals

- Theme: Houston mural photography, map discovery, neighborhood/artist/theme browsing.
- Accent: artwork color can be more expressive, but page structure stays editorial.

### Straterra

- Theme: B2B SaaS, commercial property listings, founder launch, premium real estate/investment.
- Visuals: public website and anonymized product mockups.
- Avoid: real member-only property data.

## Resource Thumbnail Brief

Purpose: make resources feel practical, designed, and worth saving.

Composition:

- Large title block.
- Tiny mono metadata.
- Simple diagram, UI fragment, or chrome object.
- Distinct but restrained accent per resource type.

Mood-board sources:

- Row 2 for systems.
- Row 3 for chrome object.
- Row 5 for section-header composition.

## Favicon Brief

Purpose: create a tiny mark that feels like Gibson Hall, not a generic initials logo.

Directions:

- `GH` monogram in brutal mono treatment.
- Chrome/signal dot or star as a tiny system marker.
- Black/paper reversible versions.

## Open Graph Brief

Purpose: make shared links feel like social thumbnails.

Composition:

- Huge editorial type.
- One chrome/cosmic object or case-specific visual.
- Small mono label: `Gibson Hall / Digital Transformation`.
- Black/white/paper foundation with one accent.
